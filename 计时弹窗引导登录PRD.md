# PRD · 微页面计时弹窗引导登录

## 1. 背景与目的

微页面（营销页面）承载着松赞的产品推广、活动引流等核心营销场景。当前未登录用户浏览微页面后直接流失，缺乏转化引导。

**目的**：在用户浏览营销页面时，通过计时弹窗在合适的时机引导未登录用户完成登录/注册，提升用户转化率与会员沉淀。

---

## 2. 业务流程

```
用户进入微页面
    │
    ▼
检测登录状态 ──已登录──▶ 正常浏览，不触发弹窗
    │
  未登录
    │
    ▼
开始计时（倒计时 = 配置秒数）
    │
    ├── 用户主动点击了登录按钮（页面内任意位置）▶ 立即跳转登录页，停止计时
    │
    ├── 用户已登录（如中途通过其他入口登录）▶ 停止计时，不弹窗
    │
    ├── 用户离开页面 ▶ 计时终止
    │
    ▼
计时结束
    │
    ▼
弹出登录引导弹窗（半屏/弹窗样式）
    │
    ├── 点击「去登录」▶ 跳转登录页
    │
    └── 点击「暂不登录」/ 点击遮罩关闭 ▶ 关闭弹窗，本次浏览不再弹出
```

---

## 3. 功能需求

### 3.1 触发条件

| 条件 | 说明 |
|------|------|
| 用户登录状态 | 未登录（未注册）时触发，已登录不触发 |
| 触发时机 | 用户进入微页面后开始计时 |
| 同一次浏览 | 弹窗关闭后，本次页面停留期间不再重复弹出 |
| 页面切换 | 用户切换到其他页面后重新进入，重新计时（视为新一次浏览） |

### 3.2 计时规则

| 配置项 | 说明 |
|------|------|
| 计时时长 | 运营可配置，单位：秒。建议默认值 15秒 |
| 配置入口 | 微页面后台编辑器，弹窗组件配置面板 |
| 计时方式 | 倒计时，不可见（用户无感知） |

### 3.3 计时中断

以下情况停止计时且不弹窗：

- 用户在计时结束前已登录（通过页面内其他入口）
- 用户点击了页面内的登录相关按钮
- 用户离开当前微页面

### 3.4 二级页面跳转处理

当用户在计时结束前从微页面跳转至其他页面时：

| 跳转场景 | 计时器处理 | 说明 |
|---------|-----------|------|
| 点击产品卡片进入详情页 | **终止计时** | 返回微页面时重新开始计时，避免返回后立即弹窗的突兀体验 |
| 点击微页面内链接进入其他微页面 | **终止当前计时** | 新微页面开始新计时 |
| 从其他微页面返回上一个微页面 | **重新计时** | 视为新访问，重新开始计时 |
| 点击外部链接（客服、帮助等） | **终止计时** | 返回微页面时重新开始计时 |

**实现逻辑**：

```
// 离开当前微页面（任何跳转）
onNavigateAway() {
  clearTimer()  // 终止计时
}

// 返回/重新进入微页面
onPageEnter() {
  if (!isUserLoggedIn() && !hasPopupShownThisSession()) {
    startTimer(config.duration)  // 重新开始计时
  }
}
```

**设计理由**：

- 避免用户在详情页停留较长时间后返回，立即弹窗的突兀体验
- 每次进入微页面都是新的浏览场景，重新计时更符合用户预期
- 实现简单，无需维护跨页面的计时状态

### 3.4 弹窗设计

#### 弹窗样式

- 半屏弹窗，从底部滑入
- 圆角顶部，**无关闭按钮**（强制用户选择「去登录」或「暂不登录」）
- 遮罩层半透明黑色，点击遮罩关闭弹窗

#### 弹窗内容（可配置）

| 模块 | 说明 |
|------|------|
| 弹窗标题 | 可配置，默认「登录享专属优惠」 |
| 弹窗描述 | 可配置，默认「登录后解锁会员价、积分抵扣、专属行程推荐等权益」 |
| 主按钮文案 | 可配置，默认「去登录」 |
| 次按钮文案 | 可配置，默认「暂不登录」 |
| 弹窗图片 | 可选，支持上传1张图片（建议尺寸 750×300px） |
| 弹窗权益标签 | 不支持，固定仅展示标题、描述、图片、按钮 |

#### 弹窗行为

| 行为 | 说明 |
|------|------|
| 点击「去登录」 | 跳转至登录页，当前页面地址作为 returnUrl 传递，登录后返回 |
| 点击「暂不登录」 | 关闭弹窗，本次浏览不再弹出 |
| 点击遮罩 | 同「暂不登录」 |

### 3.5 微页面后台配置

在微页面编辑器中，增加「登录引导弹窗」组件：

| 配置项 | 类型 | 必填 | 默认值 |
|--------|------|------|--------|
| 是否启用 | 开关 | 是 | 关 |
| 计时时长（秒） | 数字输入 | 是 | 15 |
| 弹窗标题 | 文本 | 否 | 登录享专属优惠 |
| 弹窗描述 | 文本 | 否 | 登录后解锁会员价、积分抵扣、专属行程推荐等权益 |
| 主按钮文案 | 文本 | 否 | 去登录 |
| 次按钮文案 | 文本 | 否 | 暂不登录 |
| 弹窗图片 | 图片上传 | 否 | 无 |

## 4. 前端交互细节

### 4.1 计时逻辑

```
onPageLoad() {
  if (isUserLoggedIn()) return
  startTimer(config.duration)
}

startTimer(seconds) {
  timer = setTimeout(() => {
    if (!isUserLoggedIn()) {
      showLoginPopup()
    }
  }, seconds * 1000)
}

onUserLogin() {
  clearTimer()
  // 若弹窗已显示，自动关闭
  hideLoginPopup()
}

onPageUnload() {
  clearTimer()
}
```

### 4.2 弹窗动画

- 弹出：底部滑入，动画时长 300ms，ease-out
- 关闭：底部滑出，动画时长 200ms，ease-in
- 遮罩：淡入淡出，与弹窗同步

### 4.3 登录回调

跳转登录页时携带 `returnUrl`，登录成功后回跳至原微页面，登录状态已更新，不再触发弹窗。

## 5. 数据埋点

| 事件名 | 触发时机 | 上报参数 |
|--------|---------|---------|
| `login_popup_timer_start` | 计时开始 | pageId, duration |
| `login_popup_show` | 弹窗展示 | pageId, duration, popupConfig |
| `login_popup_click_login` | 点击「去登录」 | pageId |
| `login_popup_click_cancel` | 点击「暂不登录」/ 关闭 | pageId |
| `login_popup_login_success` | 登录成功返回 | pageId, loginDuration |

---

## 6. 边界情况

| 场景 | 处理方式 |
|------|---------|
| 用户快速离开页面（计时未结束） | 不弹窗，计时清除 |
| 用户关闭弹窗后长时间停留 | 不再弹出（同一次页面生命周期内仅一次） |
| 配置时长为 0 | 进入页面立即弹窗 |
| 多个微页面之间跳转 | 每个微页面独立计时 |
| 弱网环境下计时 | 前端本地计时，不依赖网络请求 |
| 微信内置浏览器限制 | 弹窗跳转使用微信标准页面跳转方式 |

---

## 7. 非功能性需求

- 弹窗组件不影响微页面本身的加载性能和渲染速度
- 计时器在页面不可见时暂停（`onHide` 生命周期），页面重新可见时继续计时
- 兼容微信小程序 WebView 环境
