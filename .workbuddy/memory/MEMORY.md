### 四、推荐算法实现要点

**文件**: `recommend_with_groups.py`

1. **需求解析** (`parse_requirements`)
   - 人数：正则 `(\d+)\s*[人大客位]`
   - 季节：桃花/杜鹃季/夏季/亲子
   - 目的地：拉萨/林芝/波密/梅里/香格里拉/普洱/丽江
   - 类型：自由行/私享管家/主题团
   - **出行类型**（新增，2026-04-08）：情侣/家庭/闺蜜/银发
   - **带老人小孩**：识别关键词自动设置
   - **会员等级**：格桑(不打折)/绿绒蒿(95折)/雪莲(9折)/莲(85折)/金刚(85折)
   - **外籍提醒**：识别后对涉藏区域产品添加入藏函提示

2. **形态权重配置** (`PRODUCT_TYPE_WEIGHTS`)
   - 按人数动态调整权重
   - 权重越高表示越推荐

3. **展示策略** (`get_display_strategy`)
   - **按人数**：1人主题团 / 2人三种形态 / 3人主题团+私享管家 / 4人主题团 / 5人+主题团
   - **按出行类型**：情侣→私享管家优先 / 家庭→私享管家 / 银发→私享管家 / 闺蜜→私享管家+自由行

4. **评分逻辑** (`score_product`)
   - 天数匹配、形态×人数适配、季节主题匹配、目的地匹配、标签匹配
   - **出行类型匹配**（新增）：
     - 情侣：私享管家+1.2，避免单形态主题团
     - 家庭：亲子标签+1.5，私享管家+0.8
     - 银发：低海拔/度假休闲标签+1.5，深度户外-0.8
     - 闺蜜：轻户外/美食美酒/摄影标签+1.2
   - 外籍：涉藏区域添加入藏函提醒

5. **团期优先排序**
   - 有团期的产品优先展示
   - 按评分+团期双重排序

## 核价计算规则

### 一、核心公式

**打包价 = 元素总成本 / 成本率**

**成本率 = 1 - 利润率**

**利润率 = 集团分摊与息税率 + 基础成本率 - 季节成本率**

其中（由工具实际参数反推）：
- 集团分摊与息税率 = 10%（固定）
- 基础成本率 = 5%（固定）
- 季节成本率：节日 -5% / 旺季 -2.5% / 平季 0% / 淡季 +5%

对应成本率：节日 80% / 旺季 82.5% / 平季 85% / 淡季 90%

### 二、9大计价要素

| 序号 | 要素 | 计费逻辑 |
|------|------|----------|
| 1 | 酒店住宿 | 房型等级 × 房晚数（按占床/加床人数） |
| 2 | 车辆 | 优先路虎卫士（普洱用奔驰），建议2客/辆 |
| 3 | 餐饮 | 成人标准价；儿童按年龄折扣 |
| 4 | 旅行活动 | 各体验项目按实际 |
| 5 | 司机 | 配置数量 × 服务天数 |
| 6 | 管家 | 配置数量 × 服务天数；**自由行不包含管家费用** |
| 7 | 保险 | 按出行人数 |
| 8 | 零食等其他 | 旅途其他服务 |
| 9 | 定制服务费 | 超出标品变形范围时加收 |

### 三、车司管计费天数规则

**基础规则（最近机场，单程<3小时）：**

| 接送机组合 | 计费天数 |
|-----------|----------|
| 下午接机 + 上午送机 | = 行程晚数（标准） |
| 上午接机 + 上午送机 | = 行程晚数 + 0.5天 |
| 下午接机 + 下午送机 | = 行程晚数 + 0.5天 |
| 上午接机 + 下午送机 | = 行程晚数 + 1天 |

**扩展规则：**
- 单程>3小时：单程+0.5天，往返+1天（示例：保山↔石月亮）
- **自由行**：按行程天数 = 晚数+1天（标准接送机），不叠加额外天数；异地接送同样按行程天数核算
- 异地接送机（非最近机场）：视作旅游行程，按接送机时长计费
- 隔夜叠加：需+0.5天

## 库存查询 API

**接口地址**：`POST https://gds.songtsam.com/product-room/bks/productType/listPrSingleOrderRoom`

**请求参数（正确格式，2026-04-04验证）**：
```json
{
  "hotelGroupCode": "SONGTSAM",
  "otaChannel": "CRS",
  "maxMemberLevel": "002",
  "unitCode": "SONGTSAM",
  "hotelCodes": ["STLKBSC"],
  "beginDate": "2026-06-20",
  "endDate": "2026-06-21"
}
```
⚠️ 注意：日期必须用 `YYYY-MM-DD` 格式（带横线），`hotelCodes` 是数组，不是单个 `hotelCode`

**返回字段**：
- `rmtype/rmtypeDesc`：房型代码/名称
- `detailDtos[].pmsNum`：公共池可用间数（酒店总房间数）
- `detailDtos[].blockNum`：预保留间数（从公共池预先划出，用于旅行产品或单订房）
- `detailDtos[].saleNum`：预分配间数（已开团分配好团期的房间）
- `detailDtos[].costPrice`：成本价
- `detailDtos[].realPrice`：**售价**（查酒店库存时输出此字段。客房产品类型不同，realPrice 也不同，如"成本接待"产品 realPrice = 500 元，"单房含双早"产品有对应餐饮成本）
- **剩余可售 = saleNum - blockNum**（预分配内剩余间数）
- 预订顺序：优先预分配 → 预保留满则走公共池 → 公共池也无则无房

**Headers**：`Authorization: {token}`, `Content-Type: application/json`

**已实现的工具**：
- `产品推荐助手.html`：支持自然语言查询酒店库存，如"4月22日南迦巴瓦有房吗"、"梅里明晚房价"
- `LittleDorje`：https://github.com/Gakkiless/LittleDorje（GitHub Pages: https://gakkiless.github.io/LittleDorje/）
