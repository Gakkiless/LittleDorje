# 服务协议弹窗 - 产品需求文档

**版本**：V1.0 | **日期**：2026-03-12 | **平台**：微信小程序

---

## 一、需求概述

用户首次使用小程序前需阅读并同意《用户协议》《隐私政策》《会员政策》三项协议，设计交互友好的弹窗引导用户完成阅读。

## 三、功能需求

| 序号 | 功能 | 描述 |
|------|------|------|
| 1 | Tab切换 | 三个Tab（用户协议、隐私政策、会员政策），点击切换内容 |
| 2 | 滚动阅读 | 内容区支持上下滚动 |
| 3 | 自动标记已读 | 滚动到内容底部（≥90%）自动标记该协议为已读 |
| 4 | 进度提示 | 底部显示当前阅读进度（X/3），阅读完显示"已阅读完XX协议" |
| 5 | 按钮激活 | 三个协议全部阅读完成后，「已阅读并同意」按钮变为绿色可点击 |
| 6 | 不同意按钮 | 始终可点击，点击弹出提示 |

---

## 四、交互流程

```
用户进入 → 阅读协议1 → 滚动到底部 → 标记已读(1/3) 
       → 切换Tab2 → 阅读协议2 → 滚动到底部 → 标记已读(2/3)
       → 切换Tab3 → 阅读协议3 → 滚动到底部 → 标记已读(3/3)
       → 按钮激活 → 点击同意 → 记录同意时间 → 关闭弹窗
```

## 六、数据记录

用户点击「已阅读并同意」时，需上传以下数据到服务端：

| 字段 | 说明 |
|------|------|
| user_id | 用户ID |
| consent_time | 同意时间 |
| agreement_version | 协议版本号 |
